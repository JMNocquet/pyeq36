pyeq.lib.make_inversion.pyeq_nnls(ATPA, ATPB, nnls, verbose=False)[source]¶

solves the normalized linear system ATPA m = ATPB using non-negative least-squares This is a wrapper to various solvers. See notes.

Parameters

  • ATPA – m x m normal matrix as a numpy 2D array

  • ATPB – 1D numpy array of normalized right-hand-side vector

  • nnls – string, routine used for nnls. See note

  • verbose – verbose mode

Return X, time

solution vector as a 1D numpy array of lenght m, time of inversion

Note

available methods:

scipy_nnls : the classical Lawson and Hansen (1974) nnls. Slow for large system nnls_predotted: from https://github.com/nnls - Alexfields lsqnonneg: python transcription of Matlab lsqnonneg, usually very slow nnlsm_block_pivot: the preferred method on 02/03/2018 ref: J. Kim and H. Park, Fast nonnegative matrix factorization: An active-set-like method and comparisons, SIAM Journal on Scientific Computing, vol. 33, no. 6, pp. 3261-3281, 2011. nnlsm_activeset: J. Kim and H. Park, Fast nonnegative matrix factorization: An active-set-like method and comparisons, SIAM Journal on Scientific Computing, vol. 33, no. 6, pp. 3261-3281, 2011. scipy_lsq_linear: scipy linear least-squares routine scipy_lbfgsb: scipy lbfgs routine)

pyeq.lib.make_inversion.pyeq_bvls(A, B, bounds_slip, verbose=True)[source]¶

Stark & Parker algorithm Bounds are assumed to be between 0 and max_slip*m0, where max_slip is a vector of dim(m) This used to be a wrapper to use bvls.so f2py compiled from the original bvls.f routine. Looks like scipy.optimize.lsq_linear does the bvls problem.

G : Green’s functions d : Observation vector max_slip : vector of max_slip
